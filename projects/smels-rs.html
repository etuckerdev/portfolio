SMELS - Small Model Error Log Summarizer

Intelligent CLI tool that analyzes and provides actionable fixes for errors across multiple programming languages using local AI models for privacy-focused development workflow enhancement.

Developer Problem Solved

As a developer working across multiple languages and frameworks, I frequently encountered the tedious process of parsing error messages, searching documentation, and finding solutions. This time-consuming workflow interrupted development flow and reduced productivity, especially when working with unfamiliar error formats across different programming environments.

Solution & Architecture

  • Built a Rust-based CLI tool that intelligently parses error logs from multiple programming languages (Rust, JavaScript, Python, Java).
  • Integrated with Ollama for local AI processing, ensuring privacy by keeping error logs and analysis completely offline.
  • Implemented multi-format error detection with language-specific parsing strategies for accurate analysis.
  • Created a companion web interface with caching mechanisms for improved performance and user experience.
  • Published as an open-source crate on crates.io, making it easily installable for the Rust community.

Technical Implementation

  • Language Detection: Smart parsing algorithms that identify error context across different programming languages and frameworks.
  • Privacy-First Design: All processing happens locally using Ollama, ensuring sensitive code and error information never leaves the developer's machine.
  • Performance Optimization: Efficient Rust implementation with caching strategies to minimize AI model calls and improve response times.
  • Extensible Architecture: Modular design allowing easy addition of new programming languages and error formats.
