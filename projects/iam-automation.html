Enterprise IAM Automation Platform

PowerShell and Python-driven user provisioning system for 500-user organization across 450+ applications with compliance reporting for financial services regulations.

Challenge

Stax Inc. needed to manage identity and access across 450+ applications for 500+ users while maintaining compliance with financial services regulations. Manual provisioning processes were time-consuming, error-prone, and created compliance risks.

Solution

  • Developed PowerShell and Python automation scripts for user provisioning across Okta, Auth0, and Azure AD platforms.
  • Implemented automated workflow integration with Jira ticketing system and Monday.com for project tracking.
  • Built comprehensive SQL database logging system with complex stored procedures for compliance reporting.
  • Created standardized security protocols and audit procedures for regulatory adherence.

Impact

  • Achieved 30% reduction in manual provisioning processes, saving significant operational time.
  • Established automated compliance reporting pipeline meeting financial services regulatory requirements.
  • Improved security posture through standardized access control procedures and audit trails.
  • Enhanced operational efficiency with integrated ticketing and project management workflows.

Tech stack

  • PowerShell
  • Python
  • Okta
  • Azure AD
  • Auth0
  • SQL Server
  • Jira API
  • Monday.com API

Key learnings

This project demonstrated the critical importance of automation in enterprise IAM environments. Working with financial services compliance requirements taught me the value of comprehensive audit trails and the complexity of managing access across diverse application ecosystems.
